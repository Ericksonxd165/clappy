# Documentación del Backend - Proyecto Clappy

## 1. Resumen General

Este backend está desarrollado con **Django** y **Django REST Framework**. Sirve como la API para la aplicación Clappy, gestionando la lógica de negocio relacionada con usuarios, productos (cajas), pagos y notificaciones.

La autenticación se maneja a través de tokens web JSON (JWT), utilizando la librería `djangorestframework-simplejwt`.

## 2. Base de Datos

El sistema está configurado para utilizar **PostgreSQL** en producción y desarrollo. Para el entorno de pruebas (`pytest`), se utiliza **SQLite** para agilizar la ejecución.

## 3. Estructura del Proyecto

El backend se divide en dos aplicaciones principales de Django:

-   `users`: Contiene toda la lógica para el registro, autenticación, gestión de perfiles de usuario y recuperación de contraseñas.
-   `clap`: Maneja la funcionalidad principal de la aplicación, incluyendo la gestión de "cajas" (productos), el registro de pagos, el sistema de notificaciones y las configuraciones del administrador.

## 4. Documentación de Endpoints de la API

A continuación se detallan todos los endpoints disponibles.

### 4.1. Endpoints de Autenticación y Usuarios (`/users/api/v1/`)

Gestionan el ciclo de vida del usuario y la autenticación.

| Método | Endpoint                                           | Permisos      | Descripción                                                                                             |
| :----- | :------------------------------------------------- | :------------ | :------------------------------------------------------------------------------------------------------ |
| `POST` | `/register/`                                       | `Público`     | Registra un nuevo usuario en el sistema.                                                                |
| `POST` | `/login/`                                          | `Público`     | Autentica a un usuario y devuelve un par de tokens JWT (`access` y `refresh`).                          |
| `POST` | `/token/refresh/`                                  | `Público`     | Refresca un token de acceso JWT expirado utilizando un token de refresco válido.                        |
| `GET`  | `/me/`                                             | `Autenticado` | Obtiene los datos del perfil del usuario que realiza la solicitud.                                      |
| `PUT`  | `/me/`                                             | `Autenticado` | Actualiza los datos del perfil del usuario que realiza la solicitud.                                    |
| `POST` | `/password-reset/`                                 | `Público`     | Inicia el proceso de recuperación de contraseña. Envía un correo con un enlace de reseteo.              |
| `POST` | `/password-reset-confirm/<uidb64>/<token>/`        | `Público`     | Confirma y establece una nueva contraseña utilizando el token recibido por correo.                      |

### 4.2. Endpoint de Gestión de Usuarios (`/users/api/v1/users/`)

CRUD completo para la gestión de usuarios. **Solo para administradores.**

| Método   | Endpoint      | Permisos        | Descripción                               |
| :------- | :------------ | :-------------- | :---------------------------------------- |
| `GET`    | `/`           | `Administrador` | Lista todos los usuarios del sistema.     |
| `POST`   | `/`           | `Administrador` | Crea un nuevo usuario.                    |
| `GET`    | `/<id>/`      | `Administrador` | Obtiene los detalles de un usuario.       |
| `PUT`    | `/<id>/`      | `Administrador` | Actualiza completamente un usuario.       |
| `PATCH`  | `/<id>/`      | `Administrador` | Actualiza parcialmente un usuario.        |
| `DELETE` | `/<id>/`      | `Administrador` | Elimina un usuario del sistema.           |
